## Introduction

### What is CXV 3DViewer SDK?

CXV 3DViewer SDK is a lightweight, framework-agnostic JavaScript library that allows you to embed and control a 3D Viewer in any web application. It provides:

- **Easy Integration**: Works with vanilla JS, React, Vue, Angular, etc.
- **Flexible Usage Modes**: Direct viewer mode or file pipeline mode
- **Rich API**: Control camera, interaction, toolbar, model tree, and more
- **Event-Driven**: Subscribe to lifecycle and state changes
- **Lightweight**: Minimal footprint without bundling the viewer application

### Key Features

- 📦 **File Management**: Upload, convert, and render 3D files
- 🎥 **Camera Control**: Zoom, pan, orbit, and home view
- 🖱️ **Interaction Modes**: Select, area select, pan, orbit, walk-through, zoom window
- 🎨 **Drawing Modes**: Shaded, wireframe, hidden line, X-ray, ghosting
- 🔧 **Toolbar Control**: Manage panels, clipping planes, settings, model tree
- 🌳 **Model Tree**: Navigate and select nodes in the model hierarchy
- 📊 **State Management**: Track upload/conversion/rendering stages and status

## Installation & Setup

### Prerequisites

- **Node.js**: v14.0.0 or higher
- **npm**: v6.0.0 or higher (or yarn, pnpm)
- **Viewer App**: A separate 3D Viewer application must be deployed

### Step 1: Install the SDK

Using npm:

```bash
npm install 3dviewer-sdk
```

Or using yarn:

```bash
yarn add 3dviewer-sdk
```

Or using pnpm:

```bash
pnpm add 3dviewer-sdk
```

### Step 2: Set Up a Viewer Application

The SDK does NOT include the viewer application. You must deploy it separately.

#### Option A: Run Viewer Locally (for Development)

```bash
# Clone the viewer repository
git clone --branch 3dviewer_SDK --single-branch https://git.anybim.vn/construxiv/cxv-3dviewer

# Navigate to the viewer directory
cd cxv-3dviewer

# Install dependencies
npm install

# Start the viewer app
npm run start
```

The viewer will be available at `http://localhost:3000`.

#### Option B: Deploy Viewer to Production

Update your SDK configuration to point to your production viewer:

```js
const viewer = new Viewer3D({
  container: "#app",
  baseUrl: "https://api.example.com",
  viewerPath: "/mainviewer",
  allowedOrigin: "https://viewer.example.com",
});
```

### Step 3: Verify Viewer Accessibility

Open `http://localhost:3000` in your browser. You should see the viewer dashboard. Navigate to a file and copy the final `mainviewer` URL.

Example final URL:
```
http://localhost:3000/mainviewer?fileList=abc123&token=xyz789
```

## Core Concepts

### Two Usage Modes

#### 1. Direct Viewer Mode

Use when you already have a final viewer URL.

**When to use:**
- You have a pre-converted file
- The URL is generated by another service
- You want minimal SDK involvement

**Example:**
```js
const viewer = new Viewer3D({
  container: "#app",
  url: "http://localhost:3000/mainviewer?fileList=...",
  allowedOrigin: "http://localhost:3000",
});
```

#### 2. File Pipeline Mode

Use when the SDK should handle upload → conversion → open.

**When to use:**
- Users are uploading fresh files
- You want a seamless upload-to-view experience
- You need upload/conversion state tracking

**Example:**
```js
const viewer = new Viewer3D({
  container: "#app",
  baseUrl: "https://dev.3dviewer.anybim.vn",
  viewerPath: "/mainviewer",
  uploadPath: ".",
  allowedOrigin: "http://localhost:3000",
});
```

### Render Priority

When calling `viewer.render(file?)`:

1. If `url` is provided → render directly from URL
2. If `url` is missing → fallback to file pipeline (upload → convert → open)

## Basic Usage

### Minimal Setup

```html
<!-- HTML -->
<div id="viewer-container"></div>

<script type="module">
  import { Viewer3D } from "3dviewer-sdk";

  // Initialize
  const viewer = new Viewer3D({
    container: "#viewer-container",
    url: "http://localhost:3000/mainviewer?fileList=abc123",
    allowedOrigin: "http://localhost:3000",
    width: "100%",
    height: "100vh",
  });

  // Render
  viewer.init();
  viewer.render();
</script>
```

### With File Upload

```html
<!-- HTML -->
<div id="viewer-container"></div>
<input type="file" id="file-input" />

<script type="module">
  import { Viewer3D } from "3dviewer-sdk";

  const viewer = new Viewer3D({
    container: "#viewer-container",
    baseUrl: "https://dev.3dviewer.anybim.vn",
    viewerPath: "/mainviewer",
    uploadPath: ".",
    allowedOrigin: "http://localhost:3000",
  });

  viewer.init();

  // Handle file selection
  document.getElementById("file-input").addEventListener("change", async (e) => {
    const file = e.target.files?.[0];
    if (!file) return;

    try {
      // Upload, convert, and open in one call
      const result = await viewer.files.render(file);
      console.log("Viewer opened:", result.url);
    } catch (error) {
      console.error("Error:", error.message);
    }
  });
</script>
```

### Lifecycle Management

```js
// Initialize the viewer
viewer.init();

// Render the viewer
viewer.render();

// Destroy the viewer when done
viewer.destroy();
```

## API Reference

### Viewer3D Constructor

#### Options Object

```ts
type Viewer3DOptions = {
  // Required
  container: HTMLElement | string;

  // Direct mode (optional)
  url?: string;

  // File pipeline mode (optional)
  baseUrl?: string;
  viewerPath?: string;
  uploadPath?: string;
  file?: File;

  // Display options
  width?: string;                    // default: "100%"
  height?: string;                   // default: "100%"
  sandbox?: string;                  // iframe sandbox attribute
  
  // Security
  allowedOrigin?: string;

  // Notifications
  notify?: boolean | {
    success?: boolean;
    error?: boolean;
  };
};
```

#### Container

Specifies where to mount the viewer iframe.

```js
// Using CSS selector
const viewer = new Viewer3D({ container: "#app" });

// Using HTML element
const viewer = new Viewer3D({ 
  container: document.getElementById("viewer") 
});
```

#### Display Options

```js
const viewer = new Viewer3D({
  container: "#app",
  url: "...",
  width: "100%",           // Full width
  height: "800px",         // Fixed height
  sandbox: "allow-same-origin allow-scripts allow-forms",
});
```

#### Security with allowedOrigin

Restricts which origins can communicate with the viewer:

```js
const viewer = new Viewer3D({
  container: "#app",
  url: "http://localhost:3000/mainviewer?fileList=...",
  allowedOrigin: "http://localhost:3000", // Only this origin can send messages
});
```

**Important**: Always set this in production to prevent security vulnerabilities.

#### Notifications

Reserved option for notification behavior (currently not applied by SDK runtime):

```js
// Reserved config example (currently no runtime effect)
const viewer = new Viewer3D({
  container: "#app",
  notify: false,
});

// Reserved config example with object format
const viewer = new Viewer3D({
  container: "#app",
  notify: { success: true, error: false },
});
```

### Files Module

Handles file upload, conversion, and opening.

#### Methods

##### `upload(file: File): Promise<{ fileName: string; baseFileId: string }>`

Upload a file without converting.

```js
try {
  const response = await viewer.files.upload(file);
  console.log("File ID:", response.baseFileId);
  console.log("File name:", response.fileName);
} catch (error) {
  console.error("Upload failed:", error);
}
```

**Response:**
```ts
{
  baseFileId: string;        // Unique file ID
  fileName: string;          // Original filename
}
```

##### `convert(file: File): Promise<PreparedViewerData>`

Convert a file without opening.
This SDK uses single-shot conversion (no polling/retry inside `convert()`).

```js
try {
  const response = await viewer.files.convert(file);
  console.log("Viewer URL:", response.url);
} catch (error) {
  console.error("Conversion failed:", error);
}
```

**Response:**
```ts
{
  baseFileId: string;
  baseMajorRev: number;
  baseMinorRev: number;
  fileName: string;
  query: string;             // URLSearchParams string
  url: string;               // Final viewer URL
}
```

##### `prepare(file: File): Promise<PreparedViewerData>`

Combine upload + convert. Returns conversion result.

```js
try {
  const prepared = await viewer.files.prepare(file);
  console.log("File prepared:", prepared.url);
} catch (error) {
  console.error("Preparation failed:", error);
}
```

##### `open(prepared: PreparedViewerData | { url: string }): void`

Open a prepared file in the viewer.

```js
const prepared = await viewer.files.prepare(file);
viewer.files.open(prepared);
```

##### `render(file?: File): Promise<PreparedViewerData>`

Complete pipeline: upload → convert → open. Most convenient method.

```js
try {
  const result = await viewer.files.render(selectedFile);
  console.log("Now viewing:", result.url);
} catch (error) {
  console.error("Failed to render:", error);
}
```

##### `setConfig(config: Partial<FilesConfig>): void`

Update file pipeline configuration at runtime.

```js
viewer.files.setConfig({
  baseUrl: "https://new-api.example.com",
  viewerPath: "/viewer",
  uploadPath: "custom-upload",
});
```

##### `getState(): LoadStatePayload`

Get current file pipeline state.

```js
const state = viewer.files.getState();
console.log("Current stage:", state.stage);
console.log("Is loading:", state.isLoading);
console.log("Message:", state.message);
console.log("Elapsed ms:", state.elapsedMs);
```

## Event Handling

### File Upload/Conversion Events

The SDK provides granular events for tracking file operations.

#### Upload Events

```js
// Upload started
viewer.files.on.uploadStart((payload) => {
  console.log("Uploading:", payload.fileName);
});

// Upload successful
viewer.files.on.uploadSuccess((payload) => {
  console.log("File ID:", payload.baseFileId);
  console.log("File name:", payload.fileName);
});

// Upload error
viewer.files.on.uploadError((payload) => {
  console.error("Upload error:", payload.error);
});
```

#### Conversion Events

```js
// Conversion started
viewer.files.on.conversionStart((payload) => {
  console.log("Converting:", payload.fileName);
});

// Conversion successful
viewer.files.on.conversionSuccess((payload) => {
  console.log("Viewer URL:", payload.url);
  console.log("File ID:", payload.baseFileId);
});

// Conversion error
viewer.files.on.conversionError((payload) => {
  console.error("Conversion error:", payload.error);
});
```

#### Render Events

```js
// Render started
viewer.files.on.renderStart((payload) => {
  console.log("Rendering URL:", payload.url);
});

// Render successful
viewer.files.on.renderSuccess((payload) => {
  console.log("Successfully rendered:", payload.url);
});

// Render error
viewer.files.on.renderError((payload) => {
  console.error("Render error:", payload.error);
});
```

#### Load Events

```js
// File loaded in iframe
viewer.files.on.loadSuccess((payload) => {
  console.log("File loaded:", payload.url);
});

// Load error
viewer.files.on.loadError((payload) => {
  console.error("Load error:", payload.error);
});
```

#### State Change Events

```js
// State changed (uploading, converting, rendering, completed, etc.)
viewer.files.on.state((payload) => {
  console.log("Current stage:", payload.stage);
  // Possible values: 'idle', 'uploading', 'converting', 'rendering', 'completed', 'error'
  console.log("Elapsed(ms):", payload.elapsedMs);
});
```

### Complete Event Example


### Example 3: Error Handling & Retry Logic

```js
const viewer = new Viewer3D({
  container: "#viewer",
  baseUrl: "https://api.example.com",
  viewerPath: "/mainviewer",
  notify: false, // Reserved option (currently no runtime effect)
});

viewer.init();

const MAX_RETRIES = 3;
let retryCount = 0;

async function renderFileWithRetry(file) {
  try {
    retryCount = 0;
    const result = await viewer.files.render(file);
    console.log("Success:", result.url);
    return result;
  } catch (error) {
    if (retryCount < MAX_RETRIES) {
      retryCount++;
      console.log(`Retrying... (${retryCount}/${MAX_RETRIES})`);
      await new Promise(resolve => setTimeout(resolve, 2000)); // Wait 2s
      return renderFileWithRetry(file);
    }
    console.error("Failed after retries:", error);
    throw error;
  }
}

// Usage
document.getElementById("fileInput").addEventListener("change", async (e) => {
  const file = e.target.files?.[0];
  if (!file) return;

  try {
    await renderFileWithRetry(file);
  } catch (error) {
    showError(`Failed to load file: ${error.message}`);
  }
});
```

```js
    console.error("Failed to load sheets:", error);
  }

// Panel management
function toggleClippingPlanes() {

  const isOpen = document.querySelector("[data-panel='clipping']")?.classList.contains("open");
  if (isOpen) {
    viewer.toolbar.closeClippingPlanes();
  } else {
    viewer.toolbar.openClippingPlanes();
  }

```

## Best Practices

### 1. Security

**Always set `allowedOrigin`:**

```js
// ✅ Good
const viewer = new Viewer3D({
  container: "#app",
  allowedOrigin: "https://trusted-domain.com", // Specific domain
});

// ❌ Bad - allows any origin
const viewer = new Viewer3D({
  container: "#app",
  // No allowedOrigin set
});
```

### 2. Error Handling

**Always catch errors in file operations:**

```js
// ✅ Good
try {
  const result = await viewer.files.render(file);
} catch (error) {
  console.error("Failed:", error.message);
  showUserMessage("Could not load file. Please try again.");
}

// ❌ Bad - no error handling
const result = await viewer.files.render(file);
```

### 3. Memory Management

**Destroy viewer when done:**

```js
// ✅ Good
class App {
  viewerInstance = null;

  init() {
    this.viewerInstance = new Viewer3D({ container: "#app" });
    this.viewerInstance.init();
  }

  cleanup() {
    if (this.viewerInstance) {
      this.viewerInstance.destroy();
    }
  }
}

const app = new App();
window.addEventListener("unload", () => app.cleanup());
```

### 4. Event Subscription Cleanup

```js
// ✅ Good - store reference to unsubscribe later
const unsubscribe = viewer.files.on.uploadSuccess((p) => {
  console.log(p);
});

// Later, unsubscribe
unsubscribe?.();
```

### 5. Configuration Validation

```js
// ✅ Good - validate config before use
function createViewer(options) {
  if (!options.container) {
    throw new Error("Container is required");
  }

  if (!options.url && !options.baseUrl) {
    throw new Error("Either url or baseUrl must be provided");
  }

  return new Viewer3D(options);
}
```

### 6. File Type Validation

```js
// ✅ Good - validate file types
const ALLOWED_TYPES = [
  "application/zip",
  "application/x-zip-compressed",
  "model/gltf+json",
  "model/gltf-binary",
];

async function handleFile(file) {
  if (!ALLOWED_TYPES.includes(file.type)) {
    throw new Error(`File type ${file.type} is not supported`);
  }

  return viewer.files.render(file);
}
```

### 7. Loading States

```js
// ✅ Good - provide user feedback
async function renderFile(file) {
  const btn = document.getElementById("uploadBtn");
  const spinner = document.getElementById("spinner");

  btn.disabled = true;
  spinner.style.display = "block";

  try {
    await viewer.files.render(file);
  } finally {
    btn.disabled = false;
    spinner.style.display = "none";
  }
}
```

### 8. Responsive Design

```js
// ✅ Good - handle responsive viewer
const viewer = new Viewer3D({
  container: "#viewer",
  width: "100%",    // Responsive
  height: "100%",   // Responsive
});

// Listen to resize events
window.addEventListener("resize", () => {
  // Viewer should handle this automatically,
  // but you can trigger updates if needed
});
```

## Troubleshooting

### Common Issues & Solutions

#### Issue: "Viewer not loading" or blank iframe

**Possible Causes:**
1. Viewer app not running
2. Wrong URL format
3. CORS issues
4. Origin mismatch

**Solutions:**
```js
// Check 1: Verify viewer is running
// Open http://localhost:3000 in browser

// Check 2: Ensure URL is complete
const viewer = new Viewer3D({
  container: "#app",
  url: "http://localhost:3000/mainviewer?fileList=...", // Must have query params
  allowedOrigin: "http://localhost:3000",
});

// Check 3: Set correct allowedOrigin
// Must match the origin of your viewer app exactly
const viewer = new Viewer3D({
  container: "#app",
  allowedOrigin: "http://localhost:3000", // If viewer is at localhost:3000
});
```

#### Issue: "File upload fails"

**Possible Causes:**
1. Wrong baseUrl
2. Network issues
3. File too large
4. API authentication

**Solutions:**
```js
// Check 1: Verify baseUrl and paths
const viewer = new Viewer3D({
  container: "#app",
  baseUrl: "https://dev.3dviewer.anybim.vn", // Correct dev API
  viewerPath: "/mainviewer",
  uploadPath: ".",                            // Usually "."
});

// Check 2: Monitor upload lifecycle
viewer.files.on.uploadStart((p) => {
  console.log("Upload started:", p.fileName);
});

viewer.files.on.uploadError((p) => {
  console.error("Upload error:", p.error);
});

// Check 3: Validate file before upload
if (file.size > 500 * 1024 * 1024) { // 500MB limit
  throw new Error("File is too large");
}
```

#### Issue: "Conversion timeout"

**Possible Causes:**
1. File is very large
2. Conversion API is slow
3. Network timeout

**Solutions:**
```js
// Try with longer timeout
const sheets = await viewer.toolbar.getSheets({ timeoutMs: 30000 }); // 30 seconds

// Add retry logic
async function convertWithRetry(file, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await viewer.files.convert(file);
    } catch (error) {
      if (i === maxRetries - 1) throw error;
      await new Promise(r => setTimeout(r, 5000)); // Wait 5s
    }
  }
}
```

#### Issue: "postMessage errors in console"

**Cause:** Origin mismatch or security policy

**Solution:**
```js
// Ensure allowedOrigin matches viewer origin exactly
const viewer = new Viewer3D({
  container: "#app",
  url: "http://localhost:3000/mainviewer?...",
  allowedOrigin: "http://localhost:3000", // Must match exactly
});

// In production:
const viewer = new Viewer3D({
  container: "#app",
  url: "https://viewer.example.com/mainviewer?...",
  allowedOrigin: "https://viewer.example.com", // Production domain
});
```

#### Issue: "Container not found"

**Cause:** HTML element doesn't exist or not loaded yet

**Solutions:**
```js
// Check 1: Use string selector
const viewer = new Viewer3D({
  container: "#viewer", // CSS selector
});

// Check 2: Or use DOM element (after DOM is ready)
window.addEventListener("DOMContentLoaded", () => {
  const viewer = new Viewer3D({
    container: document.getElementById("viewer"),
  });
});

// Check 3: Use class selector
const viewer = new Viewer3D({
  container: ".viewer-container",
});
```

## FAQs

### Q: Can I use the SDK with React/Vue/Angular?

**A:** Yes! The SDK is framework-agnostic.

**React Example:**
```jsx
import { useEffect, useRef } from "react";
import { Viewer3D } from "3dviewer-sdk";

export function Viewer3DComponent() {
  const containerRef = useRef(null);
  const viewerRef = useRef(null);

  useEffect(() => {
    if (!containerRef.current) return;

    viewerRef.current = new Viewer3D({
      container: containerRef.current,
      url: "http://localhost:3000/mainviewer?fileList=...",
    });

    viewerRef.current.init();
    viewerRef.current.render();

    return () => viewerRef.current?.destroy();
  }, []);

  return <div ref={containerRef} style={{ width: "100%", height: "100vh" }} />;
}
```

### Q: Where should I host the viewer application?

**A:** The viewer app should be hosted separately. Options:

1. **Local development**: Run locally at `http://localhost:3000`
2. **Cloud deployment**: Host on AWS, Azure, GCP, etc.
3. **Your own server**: Deploy to your company servers

Then point the SDK to your hosted viewer:

```js
const viewer = new Viewer3D({
  container: "#app",
  baseUrl: "https://your-api-domain.com",
  viewerPath: "/mainviewer",
  allowedOrigin: "https://your-api-domain.com",
});
```

### Q: What file formats are supported?

**A:** Supported formats include:
- IFC (Industry Foundation Classes)
- STEP/STP
- IGES/IGS
- OBJ
- FBX
- GLTF/GLB
- PDF (for 2D drawings)

Check the viewer documentation for complete list.

### Q: How do I handle large file uploads?

**A:** The SDK should handle large uploads automatically, but:

```js
// Add upload/conversion monitoring
viewer.files.on.uploadSuccess((p) => {
  console.log("Upload done:", p.fileName, p.baseFileId);
});

viewer.files.on.state((p) => {
  console.log("Stage:", p.stage, "Elapsed:", p.elapsedMs);
});

// Set longer timeouts for large files
const result = await viewer.files.prepare(largeFile);

// Consider splitting very large files if possible
```

### Q: Can I customize the viewer UI?

**A:** The viewer app itself handles the UI. The SDK controls it via methods:

```js
// Control toolbar visibility
// Requires viewer-side support for `viewer-toolbar-config`
viewer.toolbar.disableAll3D();
viewer.toolbar.setDisabled3D(["pan", "zoomIn"]);

// Control panels
viewer.toolbar.closeModelTree();
viewer.toolbar.closeClippingPlanes();
```

### Q: How do I get the current camera position?

**A:** The SDK doesn't expose camera position directly, but you can:

1. Use viewer methods (zoom, home, etc.)
2. Listen to camera events
3. Implement custom camera scripts in the viewer app

### Q: What happens if the viewer origin changes?

**A:** Update the `allowedOrigin` to match:

```js
// If moving to new domain
const viewer = new Viewer3D({
  container: "#app",
  url: "https://new-viewer-domain.com/mainviewer?fileList=...",
  allowedOrigin: "https://new-viewer-domain.com", // Update this too!
});
```

### Q: How do I reset the SDK to initial state?

**A:** Destroy and recreate:

```js
// Cleanup
viewerInstance.destroy();

// Recreate
viewerInstance = new Viewer3D(options);
viewerInstance.init();
```

### Q: Can I use the SDK without a separate viewer app?

**A:** No. The viewer app is required and must be deployed separately. The SDK is just the control layer.
